编译项目出现内存溢出问题

前言

前段时间内在项目中遇到执行yarn build打包程序时出现报错，具体截图如下：

在图中有一句关键性的话，Ineffective mark-compacts near heap limit Allocation failed - JavaScript heap out of memory. Javascript 堆内存不足，这里说的javascript实际上指的就是node，node基于 V8 引擎，在后端开发中基本内存上使用没有什么限制，但是在node中通过javascript使用内存的时候只能使用部分的内存(64 位下约 1.4G，32 位下约 0.7G)，这就是为什么会造成内存泄漏了，因为当项目过大的时候，webpack在编译的时候会占用很多资源，如果超过默认大小限制的话就会出现以上截图中的问题了。

解决办法

其实 V8 引擎提供了选项让我们使用更多的内存，Node在启动的时候我们可以配置 max-old-space-size或者max-new-space-size来设置内存的大小。

1.Vue

2.React

这个脚本是来源于哪里呢，可以通过打开项目根目录中的node_modules文件夹,里面有个.bin的目录，react-scripts脚本就在这下面,我们打开这个文件会发现第一行#!/usr/bin/env node这样的代码，它的意思就是使用当前电脑的node来运行该文件中的javascript代码。我们这个时候就可以通过在这行代码后面加入node --max_old_space_size=4096来解决问题。
